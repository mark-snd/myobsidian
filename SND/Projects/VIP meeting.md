• 현재 파피루스는 **Kimi 2.6**을 사용 중

• 다양한 LLM을 함께 쓸 수 있는지 질문이 있었고, **가능하다고 답변**

• **Kimi와 DeepSeek이 서로 대화**하게 했을 때 결과가 좋았다는 언급

• Kimi 구동을 위해 **H100 장비 추가 도입 필요** 언급

• Kimi에 **Gemini Gem / Claude Project** 같은 기능도 구현하라는 지시

→ 업무별 지식·프롬프트·파일·설정을 묶는 커스텀 프로젝트/에이전트 기능

**2. LLM Wiki / 기존 작업 재활용**

  

• LLM Wiki는 새로 처음부터 만드는 것이 아니라 **기존 작업·지식·산출물을 재활용**하는 의미

• Wiki 본문에는 **하이퍼링크 포함** 필요

• 과거 버전에서는 되었는데 현재 버전에서 사라진 것으로 보이며, **기능 회귀/누락 확인 필요**

  

━━━━━━━━━━━━━━

**3. DCinside 데이터 / 자체 LLM**

  

• 회사가 자체 LLM을 만들고 있고, 여기에 **DCinside 데이터**를 활용하려는 맥락

• 파피루스로 DCinside 데이터를 돌린 결과물은 **부회장님과 전한석 이사**에게 공유하라는 지시

• 데이터 활용 시 법무·개인정보·저작권·약관 검토 필요

  

━━━━━━━━━━━━━━

**4. 커뮤니티 규제 대응 서비스**

  

• 7월 정부 커뮤니티 정책, 아마 **온라인차별금지법** 관련 언급
사용자가 글을 올리기 전 AI가 **차별·혐오·위법 소지**를 사전 검토하고 경고/수정 안내하는 서비스 아이디어

• 제가 간단히 검토한 결과, 시장성은 있음

• 단, “검열”이 아니라 **사용자 자율 수정 안내 / 운영 리스크 완화 / 컴플라이언스 보조**로 포지셔닝하는 게 좋음

  

━━━━━━━━━━━━━━

**5. 안심 Tool Calling**

  

• 기업 고객용 서비스에서 쿼리가 들어오면 **쿼리를 분할**하고

• 서로 다른 **IP address**를 통해 외부 검색하는 구조 고려

• 목적은 약관 우회가 아니라 **민감정보 보호, 보안, 가용성, 감사 가능성 확보**로 정리해야 함

  

━━━━━━━━━━━━━━

**6. 심볼리카**

  

• 회의 주제가 **심볼리카**로 전환됨

• 회사 내부 **회의록, Wiki, 심볼리카를 연계**하라는 지시

• 내부 회의록/Wiki의 지식과 맥락을 심볼리카가 활용하고, 결과물도 다시 지식자산으로 축적하는 구조

  

━━━━━━━━━━━━━━

• 회의 오디오 전사 후 Wiki를 만드는 과정에서, AI가 **내용·문장·용어를 보정**하는 기술 언급
